# RAIL Score

Drupal integration for the RAIL Score API: automated responsible AI content evaluation, compliance checking, and telemetry.

## Overview

The RAIL Score module integrates the [RAIL Score API](https://responsibleailabs.ai) into Drupal 9, 10, and 11. It evaluates content across 8 responsible AI dimensions, runs regulatory compliance checks, records structured audit logs, raises incidents when scores breach thresholds, and maintains a persistent human review queue for flagged dimensions.

Optional integration with the [Drupal AI module](https://www.drupal.org/project/ai) automatically scores every response generated by any registered AI provider.

## Features

- **Automatic evaluation on save** via `hook_entity_presave`
- **8 RAIL dimensions**: safety, privacy, fairness, transparency, accountability, reliability, inclusivity, user_impact
- **Two evaluation modes**: basic (fast, cost-efficient) and deep (per-dimension explanations and issues)
- **Domain and use-case context**: healthcare, finance, legal, code, education, general
- **Regulatory compliance checking**: GDPR, CCPA, HIPAA, EU AI Act, India DPDP, India AI Governance
- **Configurable thresholds**: publish gate and separate review queue threshold
- **Auto-unpublish** low-scoring content
- **Structured audit log**: every API request recorded with timing, score, cache status, and dimension breakdown
- **Incident tracking**: score breaches and compliance violations auto-raise incidents with unique IDs and severity levels
- **Human review queue**: per-dimension items flagged for manual review, persisted in the database
- **Dashboard**: API telemetry metrics, open incidents table, review queue table, all filterable by date range
- **Drupal AI module integration**: score AI-generated responses automatically via event subscriber and AI Automators plugin
- **Field formatter**: badge, progress bar, text, and full widget display modes

## Requirements

- Drupal ^9 || ^10 || ^11
- PHP >= 7.4 (PHP 8.1+ recommended)
- Guzzle HTTP client (included in Drupal core)
- RAIL Score API key ([get one here](https://responsibleailabs.ai))

## Installation

```bash
composer require drupal/rail_score
drush en rail_score -y
drush updatedb -y
```

Or via admin UI: `/admin/modules`, then run database updates at `/update.php`.

## Configuration

Navigate to `/admin/config/content/rail-score`.

### API Settings

| Setting | Description | Default |
|---------|-------------|---------|
| API Key | Your RAIL Score API key | Required |
| Base URL | API endpoint | `https://api.responsibleailabs.ai` |

Use the **Test Connection** button to verify the key before saving.

### Evaluation Settings

| Setting | Description | Default |
|---------|-------------|---------|
| Evaluation Mode | `basic` or `deep` | `basic` |
| Auto Evaluate | Evaluate on every node save | Enabled |
| Threshold | Minimum passing score (0-10) | 7.0 |
| Auto Unpublish | Unpublish nodes below threshold | Disabled |
| Dimensions | Which of the 8 dimensions to include | All |
| Content Types | Which content types to evaluate | None |

### Evaluation Context

| Setting | Description | Default |
|---------|-------------|---------|
| Domain | `general`, `healthcare`, `finance`, `legal`, `education`, `code` | `general` |
| Use Case | `general`, `chatbot`, `content_generation`, `summarization`, `translation`, `code_generation` | `general` |
| Include Explanations | Per-dimension text explanations in API response | Off |
| Include Issues | Per-dimension issue tags in API response | Off |
| Include Suggestions | Improvement suggestions in API response | Off |

### Compliance Settings

| Setting | Description | Default |
|---------|-------------|---------|
| Enable Compliance Checking | Run compliance check alongside eval | Off |
| Frameworks | GDPR, CCPA, HIPAA, EU AI Act, India DPDP, India AI Gov | GDPR |
| Strict Mode | Apply stricter threshold evaluation | Off |
| Compliance Incident Threshold | Raise incident when compliance score falls below this | 5.0 |

### Telemetry Settings

| Setting | Description | Default |
|---------|-------------|---------|
| Review Queue Threshold | Dimension score below which an item is added to the review queue | 2.0 |

### Drupal AI Module Integration

| Setting | Description | Default |
|---------|-------------|---------|
| Enable AI Integration | Score every response from any Drupal AI provider | Off |

## Adding the field_rail_score field

```bash
drush field:create node article field_rail_score \
  --field-type=decimal \
  --field-label="RAIL Score"
```

Or via admin UI: Structure > Content types > [Type] > Manage fields > Add field (Decimal).

## Dashboard

`/admin/reports/rail-score` shows six sections:

1. **API Usage Statistics** from the RAIL Score API account
2. **Local Statistics** (total evaluated, average score, below-threshold count)
3. **Recent Evaluations** table
4. **API Telemetry** (total requests, error rate, average duration, cache hit rate, score distribution)
5. **Open Incidents** table with Acknowledge and Resolve actions
6. **Human Review Queue** table with Mark Reviewed and Dismiss actions

All telemetry sections accept `?start=YYYY-MM-DD&end=YYYY-MM-DD` query parameters for date filtering.

## Programmatic Usage

### Evaluate content

```php
$client = \Drupal::service('rail_score.client');

$result = $client->evaluate('Your content here', [
  'mode'   => 'deep',
  'domain' => 'healthcare',
]);

if ($result && isset($result['result']['rail_score']['score'])) {
  $score = $result['result']['rail_score']['score'];
}
```

### Compliance check

```php
// Single framework
$result = $client->checkCompliance($content, ['framework' => 'gdpr']);

// Multiple frameworks
$result = $client->checkCompliance($content, [
  'frameworks' => ['gdpr', 'hipaa'],
]);
```

### Audit logger

```php
$audit = \Drupal::service('rail_score.audit_logger');

// Get aggregated metrics (optionally filtered by date range)
$metrics = $audit->getMetricsSummary($start_timestamp, $end_timestamp);

// Get open incidents
$incidents = $audit->getOpenIncidents(20, $start_timestamp, $end_timestamp);

// Acknowledge an incident
$audit->updateIncidentStatus('inc_a3f2b19c4d8e', 'acknowledged');
```

### Human review queue

```php
$queue = \Drupal::service('rail_score.review_queue');

// Get pending items
$items = $queue->pending(NULL, 50, $start_timestamp, $end_timestamp);

// Get counts per dimension
$counts = $queue->pendingByDimension();

// Mark an item reviewed
$queue->updateItemStatus('rev_c4d8e9f1a2b3', 'reviewed');
```

### Helper functions

```php
// Get stored RAIL Score for a node
$score = rail_score_get_score($node);

// Check whether a node passes the configured threshold
$passes = rail_score_passes_threshold($node);
```

## Drupal AI Module Integration

When the [Drupal AI module](https://www.drupal.org/project/ai) (`drupal/ai`) is installed and the integration is enabled in settings, the module provides two integration points:

### Event subscriber (automatic)

`AiResponseScorerSubscriber` listens to the `ai.generate_response.post` event. Every response from every provider (OpenAI, Anthropic, Gemini, etc.) is automatically scored. Low-scoring responses are flagged to the review queue and compliance incidents are raised if enabled.

The subscriber uses a `class_exists()` guard, so it has no cost when the AI module is absent.

### AI Automators intervention plugin (field-level)

`RailScoreInterventionPlugin` integrates with the AI Automators submodule. When an AI operation is configured to populate a field, this plugin scores the generated value before it is written. An optional **block on low score** setting prevents the value from being saved when it falls below the publish threshold.

## Database tables

The module creates three tables on install:

| Table | Purpose |
|-------|---------|
| `rail_score_audit_log` | Per-request telemetry: timing, score, mode, domain, dimension scores, errors |
| `rail_score_incidents` | Score breaches and compliance violations with severity and status tracking |
| `rail_score_review_queue` | Per-dimension items flagged for human review |

For existing installs, run `drush updatedb` to create the tables via `rail_score_update_9001`.

## Troubleshooting

**Connection test fails**
- Verify the API key is correct and active
- Check that outbound HTTPS to `api.responsibleailabs.ai` is allowed
- Review Drupal logs: `/admin/reports/dblog`

**Content not being evaluated**
- Confirm the content type is checked in the enabled content types list
- Confirm auto-evaluate is on
- Check that `field_rail_score` exists on the content type (optional but recommended)
- Review logs for API errors

**Slow saves**
- Switch evaluation mode to `basic`
- Disable `include_explanations` and `include_issues`
- Disable compliance checking on save
- Use the queue worker for bulk re-evaluation

**Tables missing after upgrade**
```bash
drush updatedb -y
```
